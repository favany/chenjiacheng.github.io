## 左值匹配

在使用 `like`（模糊匹配）的时候，在左边没有通配符的情况下，才可以使用索引。

在 MySQL 里，以 `%` 开头的 `like` 查询，用不到索引。

```
mysql> desc select * from user where name like '%abc'\G
*************************** 1. row ***************************
           id: 1
  select_type: SIMPLE
        table: user
   partitions: NULL
         type: ALL
possible_keys: NULL
          key: NULL
      key_len: NULL
          ref: NULL
         rows: 5
     filtered: 20.00
        Extra: Using where
1 row in set, 1 warning (0.03 sec)

mysql> desc select * from user where name like 'abc%'\G
*************************** 1. row ***************************
           id: 1
  select_type: SIMPLE
        table: user
   partitions: NULL
         type: range
possible_keys: name
          key: name
      key_len: 98
          ref: NULL
         rows: 1
     filtered: 100.00
        Extra: Using index condition
1 row in set, 1 warning (0.01 sec)

# 此处的 % 是一个普通的字符，不是通配符了。
mysql> desc select * from user where name='%abc'\G
*************************** 1. row ***************************
           id: 1
  select_type: SIMPLE
        table: user
   partitions: NULL
         type: ref
possible_keys: name
          key: name
      key_len: 98
          ref: const
         rows: 1
     filtered: 100.00
        Extra: NULL
1 row in set, 1 warning (0.00 sec)
```

比如：根据歌词搜索歌曲的名称，根据剧情搜索电影的名称，sphinx 来完成。

## OR 运算

如果出现 `OR`（或者）运算，要求所有参与运算的字段都存在索引，才会使用到索引。

```
# age 没有添加索引
mysql> desc select * from user where id=4 or age=12\G
*************************** 1. row ***************************
           id: 1
  select_type: SIMPLE
        table: user
   partitions: NULL
         type: ALL
possible_keys: PRIMARY
          key: NULL
      key_len: NULL
          ref: NULL
         rows: 5
     filtered: 36.00
        Extra: Using where
1 row in set, 1 warning (0.00 sec)

# id 和 name 都有索引
mysql> desc select * from user where id=4 or name='abc'\G
*************************** 1. row ***************************
           id: 1
  select_type: SIMPLE
        table: user
   partitions: NULL
         type: index_merge
possible_keys: PRIMARY,name
          key: PRIMARY,name
      key_len: 4,98
          ref: NULL
         rows: 2
     filtered: 100.00
        Extra: Using union(PRIMARY,name); Using where
1 row in set, 1 warning (0.03 sec)
```

```
mysql> desc select * from user where id=4 or age=12\G
*************************** 1. row ***************************
           id: 1
  select_type: SIMPLE
        table: user
   partitions: NULL
         type: ALL
possible_keys: PRIMARY
          key: NULL
      key_len: NULL
          ref: NULL
         rows: 5
     filtered: 36.00
        Extra: Using where
1 row in set, 1 warning (0.00 sec)

# age 添加索引
mysql> alter table user add index (age);
Query OK, 5 rows affected (0.14 sec)
Records: 5  Duplicates: 0  Warnings: 0

mysql> desc select * from user where id=4 or age=12\G
*************************** 1. row ***************************
           id: 1
  select_type: SIMPLE
        table: user
   partitions: NULL
         type: index_merge
possible_keys: PRIMARY,age
          key: PRIMARY,age
      key_len: 4,1
          ref: NULL
         rows: 2
     filtered: 100.00
        Extra: Using union(PRIMARY,age); Using where
1 row in set, 1 warning (0.00 sec)
```

## 多列索引

对于创建的多列（复合）索引，只要查询条件使用了最左边的列，索引一般就会被使用。

```
mysql> alter table user add index (name,age,email);
Query OK, 5 rows affected (0.05 sec)
Records: 5  Duplicates: 0  Warnings: 0

mysql> desc select * from user where email='abc'\G
*************************** 1. row ***************************
           id: 1
  select_type: SIMPLE
        table: user
   partitions: NULL
         type: ALL
possible_keys: NULL
          key: NULL
      key_len: NULL
          ref: NULL
         rows: 5
     filtered: 20.00
        Extra: Using where
1 row in set, 1 warning (0.00 sec)

# 此处用到索引是因为 name 是联合索引中最左边的列
mysql> desc select * from user where name='abc'\G
*************************** 1. row ***************************
           id: 1
  select_type: SIMPLE
        table: user
   partitions: NULL
         type: ref
possible_keys: name
          key: name
      key_len: 98
          ref: const
         rows: 1
     filtered: 100.00
        Extra: NULL
1 row in set, 1 warning (0.00 sec)
```

因为联合索引是需要按顺序执行的，比如 c1234 组合索引，要想在 c2 上使用索引，必须先在 c1 上使用索引，要想在 c3 上使用索引，必须先在 c2 上使用索引，依此。

```
mysql> desc select * from user where name='abc' and age=12\G
*************************** 1. row ***************************
           id: 1
  select_type: SIMPLE
        table: user
   partitions: NULL
         type: ref
possible_keys: name
          key: name
      key_len: 99
          ref: const,const
         rows: 1
     filtered: 100.00
        Extra: NULL
1 row in set, 1 warning (0.00 sec)

mysql> desc select * from user where name='abc' and age=12 and email='afdf'\G
*************************** 1. row ***************************
           id: 1
  select_type: SIMPLE
        table: user
   partitions: NULL
         type: ref
possible_keys: name
          key: name
      key_len: 197
          ref: const,const,const
         rows: 1
     filtered: 100.00
        Extra: NULL
1 row in set, 1 warning (0.00 sec)
```

假设某个表有一个联合索引（c1,c2,c3,c4）

```
A where c1=x and c2=x and c4>x and c3=x
B where c1=x and c2=x and c4=x order by c3
C where c1=x and c4= x group by c3,c2
D where c1=x and c5=x order by c2,c3
E where c1=x and c2=x and c5=? order by c2,c3
```

分析下面 A-E 能否可以使用索引，该表插入的数据：

```
create table t4 (
    c1 tinyint(1) not null default 0,
    c2 tinyint(1) not null default 0,
    c3 tinyint(1) not null default 0,
    c4 tinyint(1) not null default 0,
    c5 tinyint(1) not null default 0,
    index c1234(c1,c2,c3,c4)
);

insert into t4 values (1,3,5,6,7),(2,3,9,8,3),(4,3,2,7,5);
```

分析：对 name 和 age 和 email 分别建立独立索引：最终只能使用到一个索引。

```
mysql> alter table user add index (name);
Query OK, 5 rows affected (0.03 sec)
Records: 5  Duplicates: 0  Warnings: 0

mysql> alter table user add index (age);
Query OK, 5 rows affected (0.06 sec)
Records: 5  Duplicates: 0  Warnings: 0

mysql> alter table user add index (email);
Query OK, 5 rows affected (0.03 sec)
Records: 5  Duplicates: 0  Warnings: 0

mysql> desc select * from user where name='abc' and age=12 and email='afdf'\G
*************************** 1. row ***************************
           id: 1
  select_type: SIMPLE
        table: user
   partitions: NULL
         type: ref
possible_keys: name,name_2,age,email
          key: name
      key_len: 197
          ref: const,const,const
         rows: 1
     filtered: 100.00
        Extra: NULL
1 row in set, 1 warning (0.00 sec)

mysql> desc select * from user where age=12 and email='afdf'\G
*************************** 1. row ***************************
           id: 1
  select_type: SIMPLE
        table: user
   partitions: NULL
         type: ref
possible_keys: age,email
          key: age
      key_len: 1
          ref: const
         rows: 1
     filtered: 20.00
        Extra: Using where
1 row in set, 1 warning (0.00 sec)
```

如果对 name 和 age 和 email 建立了联合索引，在按照建立索引的顺序使用时，都用到了索引。

```
mysql> desc select * from user where name='abc' and age=12 and email='afdf'\G
*************************** 1. row ***************************
           id: 1
  select_type: SIMPLE
        table: user
   partitions: NULL
         type: ref
possible_keys: name
          key: name
      key_len: 197
          ref: const,const,const
         rows: 1
     filtered: 100.00
        Extra: NULL
1 row in set, 1 warning (0.00 sec)
```

结论：如果有多个条件经常出现在 where 条件中，则可以对条件字段建立联合索引。

应用：比如 goods 表里面，cat_id 和价格，就可以建立一个联合索引。
